# Slice 02 (Arc 03) — Rollup generation (plan-of-record)

> Refs: ODD-0013 §6 (generated rollup), §4.1 (next/blocked/path), §4.5
> (recomposition), §7 (`rollup` command); arc-plan D-2 (derived full-scan, not
> cached), D-3 (model in `odm-core`, CLI renders), D-4 (status vectors in
> gate-sequence order), Q-A3-1 (deferred surfacing OUT), Q-A3-2 (drift placeholder);
> ODD-0015 §1 (A3 row). `depends_on:` slice01 (tear rationale persisted, so it can be
> rendered).
>
> **Why this slice exists:** A3's headline is a single cheap view of the whole plan.
> This slice builds that view — the generated `ROLLUP.md` — from the graph. slice03
> (`orient`) and slice04 (`--json`) both *consume* this same model (D-3), so it is
> built once, here, as the single source for all three renderings.

## Goal

Build a `rollup` model in `odm-core` that assembles the whole-plan view as a **pure
function of the loaded corpus** (no I/O, no cache — D-2), and an `odm rollup` command
in `odm-cli` that renders it to `ROLLUP.md`. **Done when** `odm rollup` regenerates a
way-finding view — tree, per-node status vectors in gate order, ready/blocked, active
tears with rationale, and the provenance (origin) view — from a full scan, atomically,
and never asks to be hand-edited.

## Scope

**In:**

- **`odm-core` rollup model** (`rollup.rs`). A pure assembly over a loaded corpus +
  `GateSets`, reusing existing ops — **do not reimplement**:
  - way-finding **tree** from `Recomposition` (reverse-`part_of` forest: roots +
    children; total and unambiguous);
  - per-node **status vector** ordered by the type's gate **sequence** (`GateSet::
    sequence()`), absent gates shown as not-reached (D-4);
  - **ready** (`NodeGraph::next`) and **blocked** (`NodeGraph::blocked`, with the
    unsatisfied edges named) sets;
  - **active tears** (`NodeGraph::active_tears`) carrying their `because` rationale
    (slice01);
  - **provenance/origin** grouping (`frontmatter.origin()` → planned / discovered /
    amendment — the original-vs-emergent view, 0015 A3 / 0001-E2);
  - a **drift** slot (rendered "not yet tracked (A5)" — Q-A3-2) and a **deferred**
    slot left defined but **empty** (Q-A3-1 — no `deferred` status invented here).
- **`odm rollup` command** (`odm-cli`). Full-scan regenerate (D-2) → render Markdown →
  write `ROLLUP.md` at the repo root via odm-store (atomic write-temp-rename). The
  file carries a "generated — do not edit (`odm rollup`)" header; idempotent (same
  corpus → same bytes); `--dry-run` writes nothing.

**Rollup section order** (matches 0013 §6): way-finding tree (status vectors inline) →
ready / blocked → active tears → provenance view → drift (placeholder).

**Out:** `--json` (slice04 — pins the schema there); `orient`/`brief` composition and
bare-`odm` (slice03); deferred-node surfacing + re-entry predicate (A5, per Q-A3-1);
the `.odm/` incremental cache (A4); any `reconcile`/drift computation (A5).

## Design notes (settle here)

- **`ROLLUP.md` is a committed, generated artifact** at the repo root — the shared
  way-finding view (precedent: the legacy auto-generated `index.md`; "rollup as shared
  memory", 0015 §4), regenerated by `odm rollup`, never hand-edited. This is distinct
  from the gitignored, derived `.odm/` cache (A4). *Reversible call; flag if you'd
  rather it be gitignored.*
- The model is the **single source** for slice03 (`orient` reads it) and slice04
  (`--json` over it). Keep rendering (Markdown) in `odm-cli`; keep the assembled model
  in `odm-core` (D-3).

## Verification

`cargo test -p odm-core` + `-p odm-cli` green (model assembly + render + the `odm
rollup` command); a graph built via the CLI then `odm rollup` produces a tree +
status + ready/blocked that match `next`/`blocked`/`show`; regenerate is idempotent;
clippy `-D warnings`; no `unsafe`; coverage ≥ 90% (line) for odm-core + odm-cli. Rows
in `ledger.md`.
